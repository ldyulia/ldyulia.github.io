Xcode 11 Error: Multiple commands produce

Xcode 11로 업그레이드 직후 build failed : Multipel commands produce

iOS13이 릴리즈되고 xcode도 그에 맞춰 11로 업그레이드가 됐다

불안해서 업그레이드 바로 안 하고 싶었지만 똥멍청이같이 바로 해버렸음^^; 성질 급함^^;

그리고 아니나 다를까 업그레이드 하자마자 16개의 에러를 뱉어내며 장렬히 build failed!!

코드나 다른 세팅을 건드린 적이 없기 때문에 xcode업그레이드에 따른 에러임이 분명했다 왜냐면 그저께까지 잘 돌아갔거든ㅎ

확인을 위해 xcode - file - workspace setting에서 build system을 legacy build system으로 변경하고 다시 빌드를 해봤다. 역시 이상없이 잘 빌드됨..

ㅎ..

순간 고민했다

그냥 일단 이대로 두고 계속 작업할까ㅎㅎㅎ..

근데 어차피 언젠가는 운명에 순응하고 고쳐야 할 거 같았다 에휴

보니까 새로운 os가 릴리즈되는 시기인 가을 쯤에 대부분의 개발자커뮤니티에서 올라오는 질문들인 거 같았다

다행히 내 경우 답변은 작년 xcode10업데이트 때의 질답에서 쉽게 찾을 수 있었음 : https://stackoverflow.com/questions/50718018/xcode-10-error-multiple-commands-produce

마찬가지로 해당 타겟의 Build Phases - Copy Bundle Resources에서 문제가 된 항목들을 모두 삭제하니 다행히 정상적으로 빌드가 되었다.
