chrono file not found

chrono file not found

내가 한 거라곤… 컴포넌트 파일 하나의 위치를 변경한 것 뿐이었다..

해당 파일을 import한 위치에서 경로도 당연히 맞게 변경해주었다

근데 우리의 개복치같은 빌드시스템은 또 사정없이 피를 흘리기 시작했다… 경로 지정 다시 했다고 내가 사정사정 해도 다 필요없고 빨리 node_modules 다시 깔라고 지랄이었다.. 야박한 새끼..

그래서 불안한 가슴을 안고 결국 node_modules를 삭제 후 다시 받았다

시발…

이럴 줄 알았다

빌드 폴더도 clean해보고 캐시도 다 지워봐도 xcode에서부터 build failed이 뜨는 사태 발생 ㅠㅠ..

이번엔 무슨 chrono file not found 라는 메시지였다

나 첨봤다 진짜

ㅎ

여태 삽질하며 이런 저런 에러 메뉴를 많이도 봤지만 늘 새롭게 엿을 주는 xcode다

암튼 구글링 해봤다

당연히 바로 질문과 답이 나온다 : https://github.com/invertase/react-native-firebase/issues/1610

정확히 같은 에러 메시지였따. 세상에 나같은 멍청이들은 많다ㅎㅎ 위로가 됨

암튼 답변중에 빌드 시스템을 Lagacy로 바꿔보라기에 바꿔봤더니 됐다

근데 이건 임시방편이고.. 지난번 xcode 업데이트 후 비슷한 에러를 겪어봤기 때문에 이대로 봉합하면 안된다는 걸 알고 있었다 휴ㅜ

답변을 더 내려보니 사용하지 않는 디펜던시를 삭제했더니 해결됐다는 답이 눈에 띄었다

그러고보니 질문글의 에러메시지에도 firestore가 어쩌구 하고있네

당장 달려가서 podfile에서 firestore 삭제하고 *pod update 돌렸더니 문제 해결됐다

근데 생각해보니까 firestore의 설치 유무는 문제는 아닌 거 같고(?)

node_modules 새로 받고 pod install만 돌리고 pod update를 안 돌린게 문제였던거 같다

firebase는 pod install이 아니라 pod update를 해야 한다는 것!!!

잊 지 말 쟈 ! ! !
