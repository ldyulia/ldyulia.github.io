전역변수와 지역변수 그리고 객체지향프로그래밍까지

프로그래밍을 할 때, 같은이름의 변수 사용으로 코드가 꼬이는 경우가 빈발한다. 이를 해결하기 위해 나온 개념이 바로 전역변수와 지역변수 개념. 쉽게 말해 폴더정리를 하는 것과 같은 것이다. 같은 폴더 안에서는 같은 이름을 가진 파일이 존재할 수 없지만(덮어쓰기 되어버림), 서로 다른 폴더 안에서는 같은 이름을 가진 파일이 존재해도 상관없다. 그리고 이런 흐름으로, 문제 해결을 위해 발전해온 것이 바로 ‘객체지향 프로그래밍’이다.

스코프(scope)

scope = 유효범위 : 변수의 수명

전역변수(global) : 코드 전역에서 접근 가능한 변수

지역변수(local) : 함수 안에서 생성되고 함수 안에서만 접근 가능한 변수 (자바스크립트의 지역변수는 함수 안에서만 생성 가능하다. for문이나 if문 등에서 생성된 변수는 지역변수로 기능하지 못함)

cf) ES5에서 사용되던 var는 스코프가 함수단위이지만, ES6부터 도입된 const와 let은 스코프가 블록단위이다.

반드시 전역변수를 써야 하는 특별한 이유가 없는 한 지역변수를 써야 한다.

그러나 불가피하게 전역변수를 써야 할 경우 다음과 같이 객체를 이용해 전역변수를 만들고 관리한다.

위와 같이 코드를 작성하면 MYAPP이라는 전역변수 하나만을 생성하여 그 안에 필요한 다른 변수(=속성 : 객체 안에 들어있는 변수를 속성이라 부른다)를 지정하는 식으로 관리할 수 있다. 즉 MYAPP이라는 단 하나의 전역변수 안에 자신이 필요한 다른 전역변수들을 넣어놓음으로써 코드 충돌을 방지할 수 있는 것이다.

만약 위와 같이 MYAPP이라는 전역변수 하나 조차 사용하고싶지 않다면 아래와 같이 익명함수를 이용하여 작성하면 된다.

위와 같은 방식은 제이쿼리를 비롯한 수많은 라이브러리에서 모듈화를 시킬 때 자주 사용하는 기법이다.
